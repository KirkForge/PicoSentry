# L3-SUS-003 — Sensitive File Access

**Rule ID:** L3-SUS-003  
**Backend:** Subprocess (universal)  
**Verdict:** DENY  
**Pattern:** `(?i)(/etc/passwd|/etc/shadow|/etc/sudoers)`

## Detection

Triggers when sandboxed output references sensitive system files:

- `/etc/passwd` — System user database
- `/etc/shadow` — Password hashes
- `/etc/sudoers` — Sudo configuration

## Supply-Chain Relevance

Reading these files is a classic reconnaissance step:

- **Credential harvesting**: `/etc/passwd` reveals usernames for brute force
- **Privilege escalation**: `/etc/sudoers` reveals sudo configurations
- **Shadow file access**: `/etc/shadow` contains password hashes for cracking

## False Positives

Rare in legitimate packages. May trigger on:
- System administration tools
- User management libraries
- Security scanners (ironically)

## Mitigation

1. Block access with the `strict` or `default` policy
2. If legitimate, add specific path allowlist to policy
3. Investigate any package that reads these files
