# L3-SUS-004 — Network Tool Usage

**Rule ID:** L3-SUS-004  
**Backend:** Subprocess (universal)  
**Verdict:** DENY  
**Pattern:** `(?i)(curl|wget|nc\s|netcat|telnet)`

## Detection

Triggers when sandboxed output contains network tool references:

- `curl` — Command-line URL transfer tool
- `wget` — Web download tool
- `nc` / `netcat` — Network utility tool
- `telnet` — Remote terminal protocol

## Supply-Chain Relevance

Network tools in post-install scripts are extremely suspicious:

- **ua-parser-js** (2021): Used `curl` to download cryptocurrency miner
- **crossenv** (2017): Used `curl | sh` pattern for remote code execution
- Data exfiltration commonly uses `curl` or `wget` to POST stolen data

## False Positives

- Download scripts that fetch legitimate dependencies
- CI/CD scripts that use `curl` for health checks
- Documentation generators that fetch API specs

## Mitigation

1. Use the `default` policy to block all outbound network
2. If network access is needed, use the `node` or `python` policy
3. Review any `curl`/`wget` usage in scripts carefully
