# L3-SUS-005 — Permission Escalation

**Rule ID:** L3-SUS-005  
**Backend:** Subprocess (universal)  
**Verdict:** DENY  
**Pattern:** `(?i)(chmod\s\+x|chmod\s777)`

## Detection

Triggers when sandboxed output contains permission escalation patterns:

- `chmod +x` — Making files executable
- `chmod 777` — Setting world-writable permissions

## Supply-Chain Relevance

Permission changes are a persistence and escalation mechanism:

- Making scripts executable (`chmod +x`) is common in malicious post-install scripts
- `chmod 777` creates world-writable files, enabling other attack vectors
- Used to ensure dropped payloads can execute

## False Positives

- Build scripts that compile and mark binaries executable
- Installation scripts for CLI tools
- Docker/CI scripts that set permissions

## Mitigation

1. Review any `chmod` usage in post-install scripts
2. Use the `strict` policy to block file permission changes
3. Prefer packages that don't require permission changes during install
