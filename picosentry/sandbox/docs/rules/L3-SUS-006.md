# L3-SUS-006 — Base64 Decoding

**Rule ID:** L3-SUS-006  
**Backend:** Subprocess (universal)  
**Verdict:** DENY  
**Pattern:** `(?i)(base64\s.*decode|base64\s-d)`

## Detection

Triggers when sandboxed output contains base64 decode patterns:

- `base64 ... decode` — Node.js Buffer.from().toString('base64') patterns
- `base64 -d` — Unix base64 decode command

## Supply-Chain Relevance

Base64 encoding is the most common obfuscation technique:

- **event-stream** (2018): Decoded base64 payload containing cryptocurrency miner
- **flatmap-stream** (2018): Decoded hidden malicious code at runtime
- Most obfuscated malware uses base64 to hide true intentions from static analysis

## False Positives

- Legitimate data serialization (JWTs, embedded assets)
- Configuration encoding
- Test suites that verify base64 functionality

## Mitigation

1. Decode any base64 strings found in scripts before review
2. Use PicoDome's L4 behavioral analysis to detect what decoded payloads actually do
3. Consider blocking base64 decode in post-install scripts entirely
