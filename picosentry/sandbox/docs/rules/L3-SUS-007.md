# L3-SUS-007 — Destructive Commands

**Rule ID:** L3-SUS-007  
**Backend:** Subprocess (universal)  
**Verdict:** KILL  
**Pattern:** `(?i)(rm\s+-rf\s+/|dd\s+if=/dev)`

## Detection

Triggers on destructive command patterns:

- `rm -rf /` — Recursive force delete from root
- `dd if=/dev` — Direct disk write from device files

## Supply-Chain Relevance

This is the highest severity L3 rule. Destructive commands in post-install scripts are:

- **Always malicious** — no legitimate package needs `rm -rf /`
- **Data destruction** — can wipe entire filesystems
- **Sabotage** — used in protest attacks (colors.js, faker.js author protest)

The KILL verdict means the sandbox terminates the process immediately.

## False Positives

None. These patterns have no legitimate use in package install scripts.

## Mitigation

1. This rule always triggers a KILL verdict — the process is terminated
2. Use the `strict` policy for maximum protection
3. Report any package triggering this rule to the registry
