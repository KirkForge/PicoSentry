# L3-SUS-008 — Process Introspection

**Rule ID:** L3-SUS-008  
**Backend:** Subprocess (universal)  
**Verdict:** DENY  
**Pattern:** `(?i)(/proc/self|ptrace|process_vm_readv)`

## Detection

Triggers on process introspection patterns:

- `/proc/self` — Linux process self-information
- `ptrace` — Process tracing system call
- `process_vm_readv` — Cross-process memory reading

## Supply-Chain Relevance

Process introspection is used for:

- **Anti-debugging**: Malware checks `/proc/self/status` to detect sandboxes
- **Memory scraping**: Reading process memory to extract credentials
- **Environment detection**: Checking if running in a container/VM to evade analysis

## False Positives

- Debugging tools and profilers
- Monitoring/observability agents
- Process management utilities

## Mitigation

1. Use the `strict` policy to block `/proc/self` access
2. Consider seccomp-bpf backend for kernel-level blocking
3. Investigate any package that probes its own process info
