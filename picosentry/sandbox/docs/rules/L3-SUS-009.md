# L3-SUS-009 — SSH Key Access

**Rule ID:** L3-SUS-009  
**Backend:** Subprocess (universal)  
**Verdict:** DENY  
**Pattern:** `(?i)(\.ssh/|id_rsa|id_ed25519|authorized_keys)`

## Detection

Triggers on SSH key access patterns:

- `.ssh/` — SSH directory
- `id_rsa` — RSA private key
- `id_ed25519` — Ed25519 private key
- `authorized_keys` — SSH authorized keys file

## Supply-Chain Relevance

SSH key theft is a primary supply-chain attack goal:

- **Credential theft**: Private keys grant access to infrastructure
- **Lateral movement**: Stolen keys enable pivoting to other systems
- **Persistence**: Adding to `authorized_keys` maintains access

This is one of the most critical patterns — any package accessing SSH keys is almost certainly malicious.

## False Positives

- SSH management tools (ssh-copy-id, etc.)
- CI/CD agents that deploy via SSH
- SSH key generators (ssh-keygen)

## Mitigation

1. Never store SSH keys in project directories
2. Use the `strict` policy to block all SSH key access
3. Use SSH agent forwarding instead of key files
4. Report any package that accesses SSH keys
