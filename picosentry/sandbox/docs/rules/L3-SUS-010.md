# L3-SUS-010 — Dotfile Access

**Rule ID:** L3-SUS-010  
**Backend:** Subprocess (universal)  
**Verdict:** DENY  
**Pattern:** `(?i)(/root/|/home/\w+/\.)`

## Detection

Triggers on dotfile/dotdirectory access patterns:

- `/root/` — Root user home directory
- `/home/<user>/.` — User dotfiles and hidden directories

## Supply-Chain Relevance

Dotfile access is suspicious in package install scripts:

- **Credential harvesting**: `.gitconfig`, `.npmrc`, `.pypirc` may contain tokens
- **Configuration tampering**: Modifying shell configs (`.bashrc`, `.zshrc`)
- **Persistence**: Adding entries to `.profile`, `.bashrc` for code execution

## False Positives

- Dotfile management tools (stow, rcup, etc.)
- Shell configuration frameworks
- Home directory backup utilities

## Mitigation

1. Use the `default` or `strict` policy to block dotfile access
2. Review any package that reads from home directory dotfiles
3. Keep sensitive credentials in environment variables or vaults, not dotfiles
