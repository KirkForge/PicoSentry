# L2-ADV-001 — Advisory Vulnerability Check

**Rule ID:** L2-ADV-001  
**Severity:** HIGH  
**Confidence:** HIGH  
**Attack Vector:** Known Vulnerability Exploitation  
**Deterministic:** Yes (with same advisory DB)  

## Summary

Cross-references installed packages against known security advisories
from the OSV database, GitHub Advisory Database (GHSA), and npm advisories.
Detects packages with known CVEs or other published vulnerabilities.

## How It Works

1. Loads advisory data from the advisory database directory
   (default: `~/.local/share/picosentry/advisories/` or `$PICOADVISORY_DIR`)
2. For each installed package, queries the advisory database
3. Performs semver-aware version matching against affected ranges
4. Emits findings for packages with known, unpatched vulnerabilities

## Detection Criteria

- Package name matches an advisory entry
- Installed version falls within the advisory's affected range
- Fixed version exists and installed version is below it

## Usage

```bash
# Download advisory database
picosentry advisories fetch https://security.example.com/advisories.zip

# Scan with advisory checking
picosentry scan . --rules L2-ADV-001

# Or use bundled advisories for air-gapped scanning
picosentry scan . --rules L2-ADV-001 --use-bundled-advisories
```
