# L2-IOC-001 — Custom IoC Detection

**Rule ID:** L2-IOC-001  
**Severity:** HIGH  
**Confidence:** HIGH  
**Attack Vector:** Malicious Package Installation  
**Deterministic:** Yes  

## Summary

Detects packages that match custom Indicators of Compromise (IoCs) registered
by the organization. Teams can register known-malicious packages, vulnerable
versions, or suspicious patterns via `picosentry ioc register`.

## How It Works

1. At scan time, the rule loads all custom IoCs from the user registry
   (`~/.local/share/picosentry/ioc/`)
2. Each installed package (from `package.json` or lockfile) is matched against
   all registered IoCs
3. Matching is semver-aware: version ranges like `>=1.0.0,<2.0.0` are supported
4. A finding is emitted with the IoC metadata (attack vector, severity, references)

## Detection Criteria

- Package name matches an IoC entry
- Package version falls within the IoC's `version_range` (if specified)
- IoCs can target specific versions or all versions (`*`)

## Example Finding

```json
{
  "rule_id": "L2-IOC-001",
  "severity": "CRITICAL",
  "title": "Custom IoC Matched",
  "package": "malicious-pkg@1.2.3",
  "description": "Package matches registered IoC: Known C2 beacon — IOC-2024-001",
  "remediation": "Remove this package immediately. It matches an organizational IoC."
}
```

## Usage

```bash
# Register an IoC
picosentry ioc register --name "Known C2 beacon" \
  --package "malicious-pkg" --version ">=0.0.0" \
  --attack-vector "malware" --severity critical

# Scan with IoC detection
picosentry scan . --rules L2-IOC-001

# List registered IoCs
picosentry ioc list

# Remove an IoC
picosentry ioc remove IOC-2024-001
```
